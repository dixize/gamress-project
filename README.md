# GAMERS — сайт клуба с онлайн-бронью

Статический сайт (HTML/CSS/JS) + одна serverless-функция Vercel для приёма заявок на бронь
и отправки уведомления администратору в Telegram.

## Структура проекта

```
index.html          — вся разметка страницы
css/style.css        — стили
js/pricing.js         — тарифная сетка + расчёт цены брони
js/main.js           — рендер категорий, прайса, игр, правил
js/booking.js        — виджет бронирования (выбор места, дата/время, отправка заявки)
api/book.js          — serverless-функция: принимает бронь, шлёт в Telegram
supabase/schema.sql   — SQL-схема для БД (следующий этап, см. ниже)
images/               — фото зала
.env.example          — какие переменные окружения нужны
```

## Шаг 1 — Telegram-бот для уведомлений

1. В Telegram напиши **@BotFather** → `/newbot` → следуй инструкциям → получишь `TELEGRAM_BOT_TOKEN`.
2. Напиши что-нибудь своему новому боту (просто "привет") — это нужно, чтобы бот увидел твой чат.
3. Узнай свой `chat_id`: открой в браузере
   `https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/getUpdates`
   и найди `"chat":{"id": ...}` — это и есть `TELEGRAM_CHAT_ID`.
   (Если хочешь слать в отдельный чат/канал клуба — добавь бота туда админом и возьми id этого чата.)

## Шаг 2 — Деплой на Vercel через GitHub

1. Залей содержимое папки в свой репозиторий на GitHub (просто добавь эти файлы в корень репо).
2. На [vercel.com](https://vercel.com) → **New Project** → импортируй репозиторий. Framework preset — "Other" (это не React/Next, а чистый HTML — Vercel и так корректно раздаст статику и подхватит `/api`).
3. В **Project → Settings → Environment Variables** добавь:
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`
4. Deploy. Готово — форма бронирования сразу начнёт слать заявки в Telegram.

## Шаг 3 (опционально, следующий этап) — Supabase

Сейчас сайт не проверяет, свободно ли выбранное место — это лид-форма с мгновенным
уведомлением, финальное подтверждение брони делает администратор в Telegram.

Чтобы добавить **реальную проверку занятости мест** и **мини-админку**:

1. Создай проект на [supabase.com](https://supabase.com) (бесплатный тир достаточно для старта).
2. В SQL Editor выполни `supabase/schema.sql` — создаст таблицу `bookings`.
3. В Vercel добавь переменные `SUPABASE_URL` и `SUPABASE_SERVICE_KEY` (Project Settings → API в Supabase) — `api/book.js` уже умеет писать в неё, если эти переменные заданы.
4. Дальше нужно: (а) при загрузке страницы подтягивать занятые слоты и отмечать их в `.seat-map` как `taken`, (б) сделать `/admin.html` со списком броней и кнопками подтвердить/отменить.

Это отдельный, довольно быстрый этап доработки — скажи, когда будешь готов, соберём.

## Важные допущения, которые стоит проверить с владельцем клуба

- **Количество мест по категориям** (STANDARD 10 / POWER 8 / ULTRA 10 / EPIC-2K 2) — взято расчётно из описания зала ("общая зона на 20 ПК", "2 буткемпа по 5 ПК", "эксклюзивная комната 2 ПК"). Точную разбивку по номерам мест лучше сверить и поправить в `js/pricing.js` → `CATEGORIES`.
- **Границы вечерних тайм-слотов на выходных** — в прайсе для 3ч/5ч на выходных вечерний диапазон обрывается в 18:00/20:00 (а не идёт до утра, как в будни). Это реализовано как есть; что происходит при старте сессии позже этого времени в выходной — стоит уточнить у владельца (сейчас код подстраховывается и берёт вечернюю цену как дефолт).
- **Праздничные дни** приравниваются к тарифу выходного дня по правилам клуба — сейчас это не автоматизировано (calendar праздников не зашит), администратор должен учитывать это вручную при подтверждении.
- Промокод/скидка "Приведи друга" сейчас только текстовый блок на сайте — не встроен в расчёт цены (проверяется вручную администратором).

## Локальный запуск для проверки

Никакой сборки не требуется — можно открыть `index.html` прямо в браузере, но `/api/book`
работать не будет (это же serverless-функция Vercel). Для полной проверки с API:

```bash
npm i -g vercel
vercel dev
```
